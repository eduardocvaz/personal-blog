Você já sentiu a necessidade de uma API simples e rápida para simular um back-end durante o desenvolvimento de um aplicativo ou site? JsonServer é a resposta para os seus problemas. Neste artigo, vamos explorar como o JsonServer pode acelerar o desenvolvimento, usando um exemplo prático relacionado a entidades de um escritório. Prepare-se para descobrir como criar uma API REST completa em questão de minutos!

O que é JsonServer?

O JsonServer é uma ferramenta extremamente poderosa, mas surpreendentemente simples, que permite criar uma API REST completa com dados fictícios em JSON. É perfeito para prototipagem, testes e desenvolvimento rápido, pois elimina a necessidade de um back-end real durante a fase inicial de desenvolvimento.

Instalando o JsonServer:

Começamos pela instalação. Para fazer isso, você precisa ter o Node.js instalado. Execute o seguinte comando no seu terminal: 

npm install -g json-server

Criando um Arquivo de Dados:

A simplicidade do JsonServer é evidente no próximo passo. Crie um arquivo JSON que representará os dados do seu escritório. Aqui está um exemplo de arquivo db.json: 


  {
    "funcionarios": [
      { "id": 1, "nome": "João", "cargo": "Desenvolvedor" },
      { "id": 2, "nome": "Maria", "cargo": "Designer" },
      { "id": 3, "nome": "Pedro", "cargo": "Gerente" }
    ],
    "projetos": [
      { "id": 1, "nome": "Projeto A", "responsavel": 1 },
      { "id": 2, "nome": "Projeto B", "responsavel": 3 }
    ]
  }

Iniciando o JsonServer:

Agora, a parte mágica acontece. Execute o seguinte comando para iniciar o JsonServer: 

json-server --watch db.json

Isso inicia um servidor na porta 3000 com todas as rotas CRUD (Create, Read, Update, Delete) prontas para uso.

Acessando os Dados:

Agora, você pode acessar seus dados do escritório através das seguintes rotas:

  • /funcionarios: Obtenha todos os funcionários.
  • /projetos: Obtenha todos os projetos.

Por exemplo, para obter todos os funcionários, basta fazer uma solicitação GET para http://localhost:3000/funcionarios.

Facilidade de Uso:

A grande vantagem do JsonServer é sua facilidade de uso. Não há necessidade de escrever código de servidor complexo. Você pode criar, ler, atualizar e excluir dados com apenas algumas solicitações HTTP.

  • GET /funcionarios: Obtenha todos os funcionários.
  • GET /projetos: Obtenha todos os projetos.
  • POST /funcionarios: Adicione um novo funcionário.
  • PUT /funcionarios/1: Atualize o funcionário com ID 1.
  • DELETE /funcionarios/2: Exclua o funcionário com ID 2.

Conclusão:

O JsonServer é uma ferramenta incrivelmente útil que simplifica o desenvolvimento de APIs durante a fase de prototipagem e desenvolvimento inicial. Com uma configuração simples e uma facilidade de uso impressionante, você pode acelerar seu trabalho de desenvolvimento sem perder tempo com complexidades de back-end. Experimente o JsonServer e veja como ele pode transformar sua abordagem de desenvolvimento.

Espero que este guia prático o ajude a entender e aproveitar a facilidade do JsonServer em seus projetos de desenvolvimento de software. Boa codificação!
